# OneNode Atlas — Vision

## Explore software as a living civilization

Understanding an unfamiliar codebase is slow and fragmented. Developers jump between folders, source files, commit history, documentation, and teammates just to answer basic questions: where should I start, what does this module do, and how do the important parts connect?

OneNode Atlas transforms a repository into an interactive holographic world. Instead of treating code as a file tree, Atlas makes its structure, relationships, and history explorable. The visual world provides spatial understanding; Atlas, the AI guide, provides the explanation and learning path.

Atlas is not an IDE, a GitHub replacement, or a generic “chat with code” interface. It is a new way to understand software.

## Product promise

> In a few minutes, a developer should understand an unfamiliar repository better than they would by browsing folders alone.

## Who it is for

- Developers joining an existing project.
- Students learning from open-source repositories.
- Open-source contributors finding where to begin.
- Engineering teams explaining a system’s structure to new members.

## The Atlas oath

Every feature must answer one question:

> Does this help someone understand the software better than before?

If not, it does not belong in Atlas V1.

## Experience principles

1. **Software is a world.** A repository has structure, history, landmarks, and connections—not merely folders.
2. **Every visual means something.** Effects communicate architecture or activity; they are never decoration alone.
3. **Explore before reading.** Spatial navigation should reveal the important parts before the user is overwhelmed by detail.
4. **History is visible.** Software changes over time, and Atlas makes those changes understandable.
5. **AI guides, not overwhelms.** Atlas explains and teaches with grounded repository context.
6. **Beauty serves clarity.** Cinematic motion and holographic design make architecture easier to perceive.

## V1 identity

Atlas presents a repository as a floating holographic civilization in a dark digital void. Districts, buildings, roads, light pulses, and landmarks map to meaningful software concepts. A floating Atlas orb guides the user through a short, cinematic tour and answers questions about the repository or selected area.

The intended reaction is simple: “I have never seen a codebase explained like this—and I understand it.”
