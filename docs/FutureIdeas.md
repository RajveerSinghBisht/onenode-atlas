# OneNode Atlas — Future Ideas

This document deliberately separates valuable future work from the V1 build. None of these items should delay the core Atlas loop: analyze, explore, understand.

## Intelligence Pack

- Proactive, evidence-backed insights about architectural change, hotspots, and dependencies.
- Code quality, refactoring, maintainability, and performance guidance.
- Security analysis and dependency-risk views.
- Guided debugging/investigation workflows using logs, traces, and call graphs.
- Richer Git archaeology: explain a change using commits, pull requests, issues, and discussions.

## History and scale

- Full commit-by-commit time travel with filters and comparison views.
- Deeper symbol-level exploration: classes, functions, exports, and call graphs.
- Incremental analysis and persistent caching for very large repositories.
- Additional source control providers: GitLab, Bitbucket, and local repositories.
- Graph database/vector retrieval upgrades when repository knowledge becomes large.

## Product capabilities

- Accounts, saved Atlases, sharing, comments, and team onboarding spaces.
- IDE and browser integrations.
- Exportable architecture maps and onboarding guides.
- Search, saved journeys, and role-specific learning paths.
- Collaboration and ownership overlays.

## Presentation themes

- Alternative world themes such as floating islands, space colonies, and circuit worlds.
- Optional speech synthesis for Atlas narration.
- Richer shaders, sound ambience, and advanced cinematics after performance is proven.

## Rule for promoting an idea

An idea moves from this document into a release plan only when it has a clear user problem, reliable data source, defined interaction, and does not compromise Atlas’s core promise: helping people understand software.
